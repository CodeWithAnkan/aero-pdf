# Contributing to AeroPDF

Thank you for your interest in contributing to AeroPDF.

AeroPDF is an offline-first, privacy-focused AI PDF reader built with Flutter. The goal of this project is to create a clean, fast, ad-free, and intelligent reading experience without relying on cloud infrastructure or invasive tracking systems.

Whether you are fixing bugs, improving UI/UX, optimizing performance, or proposing new ideas, all contributions are appreciated.

---

# Before You Contribute

Please make sure to:

* Read the project documentation carefully
* Check existing issues and pull requests before creating a new one
* Keep contributions aligned with AeroPDF’s core philosophy:

  * Offline-first
  * Privacy-focused
  * Lightweight
  * User-centric
  * Minimal and practical

---

# Ways You Can Contribute

## Bug Fixes

* Fix crashes or unexpected behaviors
* Improve PDF rendering stability
* Resolve indexing or OCR issues

## Performance Improvements

* Optimize memory usage
* Improve scrolling/rendering performance
* Reduce AI processing latency

## UI/UX Enhancements

* Improve accessibility
* Refine animations and transitions
* Enhance responsiveness and usability

## Features

Examples:

* Better annotation tools
* Improved search indexing
* Additional offline AI capabilities
* Reading experience improvements

Please discuss major features through an issue before starting implementation.

# Development Setup

## Prerequisites

Make sure you have installed:

* Flutter SDK
* Dart SDK
* Android Studio or VS Code
* Android SDK
* Git

---

## Clone the Repository

```bash
git clone https://github.com/your-username/aeropdf.git
cd aeropdf
```

---

## Install Dependencies

```bash
flutter pub get
```

---

## Run the Project

```bash
flutter run
```

---

# Contribution Workflow

## 1. Fork the Repository

Create your own fork of AeroPDF.

---

## 2. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

Examples:

* feature/offline-search-improvement
* fix/pdf-render-crash
* ui/settings-redesign

---

## 3. Commit Changes

Write clear and meaningful commit messages.

Example:

```bash
git commit -m "Improve OCR indexing performance"
```

---

## 4. Push Your Changes

```bash
git push origin feature/your-feature-name
```

---

## 5. Open a Pull Request

Describe:

* What you changed
* Why the change is needed
* Screenshots/videos if UI related

---

# Code Style Guidelines

Please follow these principles:

* Keep code modular and readable
* Avoid unnecessary dependencies
* Prefer clean architecture patterns
* Write meaningful variable and function names
* Keep the UI minimal and accessible
* Optimize for low-resource devices whenever possible

---

# Privacy & Security Expectations

AeroPDF strongly prioritizes user privacy.

Contributions must:

* Avoid telemetry or trackers
* Avoid unnecessary network calls
* Respect offline-first principles
* Never compromise local user data

---

# Reporting Issues

When reporting a bug, include:

* Device information
* Android/iOS version
* Steps to reproduce
* Expected behavior
* Screenshots/logs if available

---

# Feature Requests

Feature suggestions are welcome.

Please ensure the proposal aligns with:

* Offline usability
* Performance efficiency
* Minimal user friction
* Privacy-first architecture

---

# Community Guidelines

Please be respectful and constructive in discussions and reviews.

AeroPDF is built for learning, experimentation, and creating genuinely useful software experiences.

Every contribution matters.
